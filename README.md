# GitHub User Finder (Поиск пользователей GitHub)
**Автор:** Паритова Марьяна
**Дата сдачи** 02.05.2026

## **Описание**

*GUI‑приложение для поиска пользователей GitHub с возможностью сохранения избранных профилей. Программа использует GitHub API для получения информации о пользователях и сохраняет избранные аккаунты в JSON‑файл (`izbr.json`).*

## Требования для запуска

Что нужно установить на компьютер: * Python 3.6 или выше * Библиотеки: `pip install requests`

## Как запустить

```bash 
git https://github.com/maryanaparitova-cyber/User-Finder
cd User-Finder 
python main.py
```

## Как пользоваться

1.  При запуске откроется окно приложения с полем ввода и кнопками.  
2.  Введите имя пользователя GitHub в поле «Введите имя пользователя GitHub:». 
3.  Нажмите кнопку **«Найти»** — в списке ниже отобразятся результаты поиска (логин, имя, количество публичных репозиториев).
4.  Чтобы добавить пользователя в избранное:
    
    -   выберите его в списке результатов;   
    -   нажмите кнопку **«Добавить в избранное»** — пользователь появится в списке избранных внизу окна.
        
5.  Чтобы удалить пользователя из избранного:
    
    -   выберите его в списке избранных;  
    -   нажмите кнопку **«Удалить из избранного»** — пользователь будет удалён из списка и сохранённых данных.
        
6.  Избранные пользователи автоматически сохраняются в файле `izbr.json` после каждого изменения.

## Пример работы

**Сценарий: поиск и работа с несколькими пользователями**

1.  **Поиск первого пользователя**
    
-   В поле ввода пишем: `torvalds` (Линус Торвальдс)
-   Нажимаем **«Найти»**
-   В списке результатов появляется:  
    `torvalds - Linus Torvalds - 12 репозиториев`
    
2.  **Добавление первого пользователя в избранное**
    
-   Выделяем строку с `torvalds` в списке результатов
-   Нажимаем **«Добавить в избранное»**
-   Появляется сообщение: «torvalds добавлен в избранное!»
-   Пользователь `torvalds` появляется в списке «Избранные пользователи:» внизу окна

3.  **Поиск второго пользователя**
  
-   Очищаем поле ввода
-   Вводим: `defunkt` (один из ранних разработчиков GitHub)
-   Нажимаем **«Найти»**
-   В списке результатов появляется:  
    `defunkt - Chris Wanstrath - 98 репозиториев`
    

4.  **Добавление второго пользователя в избранное**
    

-   Выделяем строку с `defunkt` в списке результатов
-   Нажимаем **«Добавить в избранное»**
-   Появляется сообщение: «defunkt добавлен в избранное!»
-   Пользователь `defunkt` добавляется в список избранных (теперь в списке два пользователя):
    
    ```
    torvalds
    defunkt
    ```

5.  **Проверка сохранения в файле**
    
-   Закрываем приложение
-   Открываем файл `izbr.json` — видим:
    
    json
    
    ```json
    [
      "torvalds",
      "defunkt"
    ]
    ```
    
6.  **Повторный запуск приложения**
    
-   Запускаем приложение заново: `python main.py`
-   Список «Избранные пользователи:» автоматически заполняется данными из `izbr.json`:
    ```
    torvalds
    defunkt
    ```
    
7.  **Удаление одного пользователя из избранного**
    
-   Выделяем `torvalds` в списке избранных
-   Нажимаем **«Удалить из избранного»**
-   Появляется сообщение: «torvalds удалён из избранного!»
-   Пользователь `torvalds` исчезает из списка избранных
-   В файле `izbr.json` остаётся только:
    
    json
    
    ```json
    [
      "defunkt"
    ]
    ```
    
8.  **Поиск пользователя с кириллическим именем**

-   Вводим: `ivanov-petr` (гипотетический пользователь)
-   Нажимаем **«Найти»**
-   Если пользователь существует, в результатах появится:  
    `ivanov-petr - Иван Петров - 5 репозиториев`
-   Если пользователя нет, появится сообщение: «Пользователь не найден (код: 404)»
    
9.  **Попытка добавить пользователя без предварительного поиска**

-   Пытаемся нажать **«Добавить в избранное»**, не выполнив поиск и не выбрав пользователя в списке результатов
-   Появляется предупреждение: «Выберите пользователя из результатов поиска!»
    

10.  **Последовательный поиск нескольких пользователей**
    

-   Вводим: `microsoft`
-   Нажимаем **«Найти»** → видим информацию о корпоративном аккаунте Microsoft
-   Вводим: `google`
-   Нажимаем **«Найти»** → видим информацию о корпоративном аккаунте Google
-   Оба результата отображаются в списке последовательно (последний результат заменяет предыдущий)

----------

**Ключевые моменты, демонстрируемые в примере:**

-   корректная работа с сохранением/загрузкой данных между запусками;
    
-   обработка разных типов пользователей (индивидуальные разработчики, корпоративные аккаунты);
    
-   устойчивость к некорректным действиям пользователя;
    
-   автоматическое обновление интерфейса при изменениях;
    
-   надёжное сохранение данных в JSON‑формате.
  

## Тестирование

| Тип теста  | Пример | Результат |
|-----|----------|---------------------|
| Позитивный | Ввод существующего пользователя (`torvalds`) | ✅ В списке результатов отображается информация о пользователе|
| Негативный | Ввод несуществующего пользователя (`nonexistentuser12345`) | ❌ Сообщение: «Пользователь не найден (код: 404)»|
| Негативный | Пустое поле поиска и нажатие «Найти» | ❌ Сообщение: «Поле поиска не должно быть пустым!»|
| Граничный | Добавление пользователя, уже находящегося в избранном | ❌ Сообщение: «[username] уже в избранном!»|
| Позитивный |Удаление пользователя из избранного | ✅ Пользователь удаляется из списка и файла `izbr.json`|
